Why

逐字稿不該「全部混在一起」

一般轉錄工具 — 沒有 speaker，一整段連在一起
我們先對齊Q2上線的範圍好那聲紋流程寫好了嗎寫好了下週可以測那這條誰來收我來吧對了客戶demo改到週四艾瑞克你有空嗎應該可以…
→ 誰說的？誰答應的？事後對不回去。

◉
具名 speaker，Action Items 直接對到人

pyannote.ai voiceprint 比對團隊聲紋庫，逐字稿直接標人名（外部來賓保留編號、不硬猜）。有了人名，會後可直接開單、assign 任務。

Aa
詞彙表 priming

人名、產品名、術語餵給 ASR，專有名詞不再聽錯（Erica → Eric）；用繁體撰寫同時把輸出偏向繁體。

⊕
官方對齊 + 幻覺標注

採 pyannote 官方 segment-level max-overlap 合併；無語音區段的 ASR 幻覺標 ⚠ 供清理，不無聲吞掉。

∞
長音檔自動處理

超過 API 上限自動切段轉錄，段與段之間傳遞上下文；詞彙表在每段都優先保留。

How it works

兩路並行，max-overlap 對齊

/tomoaid:voiceprint-setup

一段多人錄音，建好整個團隊的聲紋庫

STEP 1
丟一段會議錄音

大家都有發言的多人會議最好，一次建立多位成員。

STEP 2
自動切出樣本

diarization 分段，每位 speaker 拼接最長片段（≤ 29 秒）。

STEP 3
網頁試聽標人名

本地標記介面逐段試聽、填英文名；非成員勾略過。名字一定由人來標。

STEP 4
寫入聲紋庫

voiceprints.json，--merge 增量更新、不動既有成員。

SPEAKER_01 · 總發言 42s · 樣本 28.6s

☐ 略過
何時重跑：新成員加入 · 連續被標 UNKNOWN（聲紋失準） 隱私：voiceprint 是不可逆 feature vector，無法還原成原音
/tomoaid:meeting-notes

之後每場會議，一句話

› /tomoaid:meeting-notes ./recordings/2026-06-11.m4a

✓ 會議記錄  meetings/2026-06-11-q2-kickoff.md
✓ 逐字稿    meetings/transcripts/…-transcript.md
✓ 音檔      meetings/recordings/…m4a （自動 gitignore）

⚠ 不確定的點：
  - 12:34 處聽不清，已標 […]
  - 「待指派 Owner」共 2 條，請決定 owner
  • 摘要與分節依內容判斷，不硬套模板；沿用專案既有會議記錄慣例。
  • Action Items 不外推 owner——沒明確指派就進「待指派 Owner」，不替團隊做分工決定。
  • 清理是去蕪存菁，不是壓縮——保留所有資訊、數字與觀點；不確定的點統一列給你 review。
  • 腳本失敗就停，不憑檔名腦補會議內容；pyannote 中途掛掉自動降級為 plain 逐字稿並說明。
Privacy & Setup

上手前要知道的事

隱私
›音檔會離開你的機器：轉錄送 OpenAI、diarization 送 pyannote.ai（暫存 48 小時自動刪除）。敏感會議自行評估。
›Voiceprint 不可逆，但屬生物特徵資料——存私有 repo，取得成員同意後再建立。
›音檔不進 git：輸出時自動加入 .gitignore，只留在本機；逐字稿與會議記錄是文字檔，可搜尋、可 review。
安裝 /plugin marketplace add tomoaid/claude-plugin /plugin install tomoaid@tomoaid
›需求：OPENAI_API_KEY · PYANNOTEAI_API_KEY · ffmpeg · Python 3.10+（免 pip install）
›團隊用：寫進專案 .claude/settings.json，隊友 clone 即自動啟用（project scope）。
›第一次先跑 voiceprint-setup 建聲紋庫，之後每場會議一句 meeting-notes。
